U1 &CAUSE&                                                                 
AS Il existe au moins un objet de gestion qui comporte des caractéristiques
   de référence dont les valeurs ne peuvent pas être converties. Le journal
   complet du programme de conversion (RNGC_CONV_REF_CHAR) donne de plus   
   amples informations. Les ID des objets de gestion sont indiqués par des 
   messages d'avertissement.                                               
U1 &SYSTEM_RESPONSE&                                                       
AS Les valeurs de caractéristiques de référence de ces objets de gestion ne
   peuvent pas être copiées dans la table AUSP. Elles sont alors ignorées. 
U1 &WHAT_TO_DO&                                                            
AS Si les valeurs de caractéristiques de référence de ces objets de gestion
   ne sont pas tenues d'être copiées dans AUSP, cette action n'est alors   
   pas requise. Dans le cas contraire, les erreurs de conversion de type   
   doivent être examinées (astuce : vérifiez le type actuel de la          
   caractéristique de référence et le type actuel de la zone référencée    
   dans la table référencée : ils doivent correspondre). Corrigez les types
   de caractéristiques qui ne correspondent pas, puis exécutez de nouveau  
   la conversion.
